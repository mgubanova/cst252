Lab 4: Pseudocoding and Problem-Solving

The purpose of this lab was to teach us the basics of problem solving and how to pseudocode a program before jumping into coding.

Challenges

Both tasks were pretty easy and we enjoyed completing them.

The only challenge we had is deciding which steps are high-level and which are low-level, and what steps should we include to avoid going into the weeds. We did our best to look at the action lists from the point of view of someone who has to learn how to do the task.

For the computer task, we were unsure if we wanted it to be a two-player game or a one player game where a human is playing with the computer. We decided to go with the first option. Task 1. Framing a piece of art

  1. Measure the artwork
    • Length
    • Width
  2. Get supplies
    • Pieces of wood for a frame. (sufficient for the size of the artwork)
    • Piece of glass for a frame. (sufficient for the size of the artwork)
    • Adhesive to glue frame together
    • Screw hooks for back wiring
    • Backing paper. (sufficient for the size of the frame)
    • Wiring to hang the framed picture
    • Cardboard backing. (sufficient for the size of the artwork)
  3. Build the frame
    • Glue the four pieces of wood together, making the frame
    • Cut the glass to the appropriate size so that it fits into the frame
    • Place glass in frame
    • Prepare the artwork to be hung
    • Place artwork in frame behind glass
    • Place cardboard backing behind artwork
    • Adhese backing paper to the back of the wooden frame
    • Drill hooks into the back of the frame at an equal height
    • Tie wiring to the hooks allowing from the frame to be hung on a wall

Task 2. Creating a chess game

  1. Print the board and pieces before each game
  2. Prompt user input
  3. Receive user input
  4. Validate the input
    • Check the piece and its parameters
    • Compare to the piece moving rules
  5. Deny the input (repeat step 2), or
  6. Accept the input
    • Send input data to the opponent
    • Update printed cells to reflect the new positions of the chess piece
    • Test for win condition
  7. Repeat steps 2-5 until win condition is met
  8. Prompt the winning user side of the board with confetti!
